# Guide: Keeping Your AI Context Up to Date

> **What this is:** A practical guide to knowing when your context files need refreshing and how to do it.
>
> **Who it's for:** The person responsible for maintaining your business's AI context — typically whoever ran the initial setup.

---

## Why this matters

Your AI context files are a snapshot of your business at the time they were created. As your business changes — new products, new positioning, new team structure, updated brand voice — the context files drift out of date.

An outdated context file is worse than no context file: the AI confidently gives answers based on information that's no longer true.

Keeping context current is a maintenance task, not a one-time project.

---

## What triggers a refresh

### Immediate updates required

These changes mean a context file is now inaccurate — update as soon as possible:

| Change | File to update |
|--------|---------------|
| New product or service launched | `PRODUCTS_SERVICES.md` |
| Product discontinued, renamed, or significantly changed | `PRODUCTS_SERVICES.md` |
| Pricing changed | `PRODUCTS_SERVICES.md` |
| A claim in a context file turns out to be wrong | Whichever file contains it |
| New major competitor entered the market | `COMPETITORS.md` |
| Business significantly repositioned or pivoted | `BUSINESS_PROFILE.md`, `COMPETITORS.md` |
| AI policy updated (new approved tools, new rules) | `AI_POLICY.md` |

---

### Scheduled review triggers

These are worth checking on a regular cadence even if nothing obvious has changed:

| Cadence | What to review |
|---------|---------------|
| **Quarterly** | `PRODUCTS_SERVICES.md` — are features, pricing, and positioning still accurate? |
| **Every 6 months** | `AUDIENCE.md` — have your target customers or their pain points shifted? |
| **Annually** | All files — full review against current business reality |
| **After any brand refresh** | `BRAND_VOICE.md` — regen if tone or language changed materially |
| **After any significant business event** | Funding, acquisition, pivot, leadership change — review `BUSINESS_PROFILE.md` |

## How to update a file

### Step 1: Identify what's changed

Be clear about what specifically needs to change before opening the wizard. Vague updates ("things feel a bit different") produce vague files. Specific changes ("we launched X product at Y price, and dropped Z") produce accurate files.

### Step 2: Re-run the relevant prompt

Go to the [setup wizard](index.html) and start from the file you need to update. You don't need to redo everything — jump directly to the file that needs refreshing.

If you have the AI tool already set up:
1. Open your AI tool with the existing context file loaded (or paste it in)
2. Tell it: "I need to update this file. Here's what's changed: [describe changes]"
3. Have it update the relevant sections
4. Review the output and save the updated file

Alternative — regenerate from scratch:
1. Run the same prompt used to generate the file originally (in `TASK_LIBRARY.md`)
2. Answer the AI's questions with current, accurate information
3. This gives you a clean updated version

### Step 3: Replace the file

- Replace the file in your `your-business/context/` (or governance/ or prompts/) folder
- If using a shared cloud folder (Google Drive, OneDrive), update the file there
- If the file is uploaded to an AI tool's knowledge base, delete the old version and upload the new one

### Step 4: Update the Master Prompt (if needed)

If you updated `BUSINESS_PROFILE.md`, `BRAND_VOICE.md`, `AUDIENCE.md`, or `PRODUCTS_SERVICES.md`, regenerate your `MASTER_PROMPT_inline.md`. The inline version contains these files' content directly and will be stale if you don't.

---

## The Master Prompt specifically

The `MASTER_PROMPT_inline.md` is generated by combining your context files. It gets stale as files change. Keep it fresh by regenerating it after significant context updates.

The `MASTER_PROMPT_files.md` (short version for tools with file upload) references files by name — it doesn't need to be regenerated unless the list of files it references changes.

## Signs your context is out of date

Watch for these signals during AI use:

- The AI describes a product or feature that was changed or discontinued
- The AI uses a tone or language that doesn't match your current brand
- The AI references an old positioning ("we're the only X that does Y" — if that's no longer true)
- Team members ask "why is the AI saying this?" and the answer is "that used to be true"
- The AI mentions competitors in proportions that don't match your current market view

Any time the AI output prompts a correction, ask: is this a prompt quality issue, or does the underlying file need updating?

---

## Who owns this

Designate someone as the owner of the AI context files. This doesn't need to be a full-time job — it's a maintenance responsibility, like owning the brand guidelines or the employee handbook.

The owner's responsibilities:
- Review files on the cadence above
- Update files when the business changes
- Push updates to shared storage and notify the team
- Field questions from team members about what's in the files

Document who this is in your `AI_POLICY.md`.

---

## Tips for sustainable maintenance

**Keep the originals somewhere accessible.** Store your `your-business/` folder in Google Drive or OneDrive — a shared location, not just one person's machine.

**Note what's factual in the file.** When you write something like "we have X customers" or "our pricing starts at £Y", make a note (in a comment or a separate document) of where that fact came from. Fact sources make updates faster.

**Update files after major meetings.** Quarterly planning, pricing reviews, and brand refreshes are natural triggers. Build it into the wrap-up of those meetings.

**The cost of not updating is real.** Stale AI context produces content that's wrong, off-brand, or embarrassing. The few minutes it takes to update a file is worth it.
